## 구현 추가사항 12월 1주차 (11.29 ~ 12.05)

### 1. 프로필 수정 구현
- 계정 변경사항 DB 반영 및 이미지 첨부 가능

### 2. 다른 알바 보기 구현
- 근로자가 여러 업장에서 근무하는 것을 고려해 **2개 이상의 알바 등록 가능**하도록 구현
- 다른 알바를 선택하면 메인화면에도 해당 알바의 근로정보에 맞게 정보가 변경되도록 구현

### 3. (근로정보 수정) – 주간 근무 스케줄 기입 기능 구현
- 근무하는 스케줄을 등록하여 **근로관리 캘린더에 바로 시각적으로 보이도록** 주간 근무 스케줄 입력 폼 구현
- 단, 캘린더에 반영되는 것 외에  
  - 근무스케줄에 따라 근로시간이 자동 반영되는 기능  
  - 근무현황 등이 변경되는 기능  
  은 아직 구현하지 못함 

### 4. 캘린더에서 근무한 시간 변경 기능 구현
- 근로시간 변경, 휴가·병가 등의 사유를 반영할 수 있도록 캘린더에서 날짜 클릭 후 근무시간을 변경할 수 있도록 구현
- 단, 근로시간을 변경하더라도  
  - 근무현황  
  - 근로시간 자동 적용  
  등은 아직 구현하지 못함 

### 5. 유저별 서류관리 기능 백엔드 구현(documents 폴더)
- 각 회원별 서류 등록 구현 - 계약서, 불만사항 등 근로 관련 서류
  - 서류 등록은 가능하지만 제거 및 업데이트 보완 필요
  - 프론트부분은 의도대로 작동하지 않아 추후 보완 


## 구현 추가사항 12월 2주차 (12.06~12.12)

### 12월 1주차 개선사항
  - 근무스케줄에 따라 근로시간이 자동 반영되는 기능  
  - 근무현황 등이 변경되는 기능  
  - 근무현황/근로시간 자동 적용  

구현 완료

### 1. 캘린더 근로취소 기능 수정
- 캘린더에서 **근로기록 취소(근로날짜 삭제)**가 정상적으로 동작하지 않던 문제 수정
- 근로취소 시
  - 캘린더 주황색 표시 해제
  - 오른쪽 통계 영역(총 근로시간 / 급여 / 주휴수당 등)이 **자동으로 반영**되도록 개선

### 2. (근로관리 / 근로정보 수정) 주간 근무 스케줄 시간 입력 UX 개선
- 주간 근무 스케줄 시간 입력을 **30분 단위로 선택 가능**하도록 변경
- 시간 입력 UI를 더 **깔끔하고 직관적인 형태**로 개선하여 가독성 및 조작성 향상

### 3. 알바 추가 기능 디버깅
- 알바 등록이 정상적으로 되지 않던 문제에 대해 **원인 분석 및 수정**
- 알바 추가 후
  - 알바 목록
  - 현재 선택된 알바 상태  
  가 올바르게 갱신되도록 개선

### 4. 주휴수당 로직 및 주별 주휴수당 계산 구현
- **한국 근로기준법 기준**으로 주휴수당 대상 여부 판단 로직 구현
- 주간 근무 스케줄 및 실제 근로기록을 기준으로
  - 주별 주휴수당 계산
  - 사이드바에 **이번 주 주휴수당 표시**
  가 가능하도록 구현

### 5. 퇴직금 로직 구현
- **한국 근로기준법 기준** 퇴직금 계산 로직 구현
- 단, 퇴직금 계산 로직은
  - 평균임금 산정 방식
  - 예외 케이스(통상임금 비교 등)
  에 대해 **추가 개선이 필요한 상태**

## 근로서류 기능 추가사항

### 6. 근로서류 업로드 및 통합 관리 기능 구축 (백엔드 / 프론트엔드)
- 근로서류 페이지에서 **서류 업로드 및 통합 관리 UI/기능** 구축
- 사용자별 서류 목록 조회 및 업로드 흐름을  
  **프론트엔드 – 백엔드 연동 구조**로 구현

### 7. 근로서류 작성 백엔드 기능 구축
- 근로서류를 단순 파일 업로드가 아닌,
  **작성 및 생성 가능한 형태로 확장할 수 있도록 백엔드 기능 구축**
- 서류 템플릿 및 작성 데이터 저장 등  
  **추후 기능 확장이 가능하도록 구조 설계**
